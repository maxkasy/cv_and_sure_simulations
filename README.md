# Numerical experiments — CV and SURE

Code and outputs for the numerical-experiments section of the paper. The
experiments validate the paper's key approximations and the convergence of the
risk function. They study five designs, all in dimension `p = 10`: the four
combinations of the two leading losses (linear, logistic regression) with the two
penalties (Ridge, Lasso), plus an anisotropic Ridge stress test.

## Quick start

```
pip install numpy pandas matplotlib          # the only dependencies
python run_parallel.py                        # reproduce everything (uses all cores)
```

This runs every `(design, n)` cell, then computes the limiting risks, the
Figure-2 slice, and renders both tables and both figures into `output/` (and the
two figures into `../Figures/`). Replication counts and sample sizes live in
`run_all.CFG`. The expensive leave-one-out designs (D2–D4) dominate the runtime;
lower `reps_generic` in `CFG` for a quick look.

To explore interactively instead, open **`cv_sure_simulations.ipynb`** and run it
top to bottom (with `RECOMPUTE = False` it renders the committed results in
`output/` instantly).

## Files

| File | Role |
|---|---|
| `cvsure_sim.py` | The library: penalties (`Ridge`, `Lasso`), models (`LinearModel`, `LogitModel`), `sure`, leave-one-out `cv_and_path`, `limit_experiment`, the per-replication metrics `one_replication`, and `make_designs`. Edit `make_designs()` to change the designs. |
| `run_all.py` | Configuration (`CFG`), the single-process sweep (`main`), and all renderers (`render_designs_table`, `render_table`, `render_fig1`, `render_fig2`). `python run_all.py` runs the whole thing in one process. |
| `run_parallel.py` | Parallel driver (one worker process per `(design, n)` cell) — the recommended way to reproduce everything. |
| `chunk.py` | Restartable CLI (`cell` / `limit` / `slice` / `render`) that accumulates replications to disk — handy for topping up the expensive designs one cell at a time. |
| `cv_sure_simulations.ipynb` | Narrative notebook. |
| `output/` | Generated artifacts: `table_designs.tex`, `table_main.tex` / `.csv` (both `\input` by the manuscript), `metrics_long.csv`, `results.json`, `fig1_convergence.{pdf,png}`, `fig2_riskfunction.{pdf,png}`, and the raw per-cell `*.npz`. |

The two LaTeX tables are generated by the code and pulled into the manuscript with
`\input`, so re-running refreshes the paper's tables (and figures) in place.

## The four diagnostics (designs in rows, results in columns)

These use the same notation as the manuscript. The column name written to
`metrics_long.csv` / `results.json` is given in parentheses.

- **Δ_IF** (`Delta_IF`) — Lemma 2 (influence-function approximation); vanishes
  like `n^{-1}`.
- **Δ_CV** (`Delta_CV`) — Lemma 4 (CV ≈ SURE, recentered to absorb the constant
  `c_n`); vanishes like `n^{-1/2}`. `SURE` follows the manuscript definition,
  which carries an overall factor `½` so that it estimates the risk
  `E[½‖θ̂^λ−θ₀‖²]` on the same (loss) scale as `n`-fold CV; CV is therefore
  compared to `SURE` directly.
- **Δ_tune** (`Delta_tune`) — Lemma 5 (CV- vs SURE-tuning); relative excess loss,
  ≈ 0.
- **Δ_R = |R_n − R|** (`R_n`, `R`, `Delta_R`) — Theorem 1 / Corollary 2 (risk
  convergence).


## Reproducibility

All randomness is seeded from `CFG['seed']`. All five designs use `p = 10`; the
diagnostics table is reported at `n = CFG['n_report']` (default 400) and every
design is run to `n = 800`. Cells accumulate into `output/cell_<KEY>_<N>.npz`, so a
run can be paused, resumed, or extended:

```
python chunk.py cell D4 800 200     # add 200 more reps to the D4, n=800 cell
python chunk.py render              # rebuild tables + figures from all cells
```
